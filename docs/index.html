Kashiwazaki SEO Author Schema Display マニュアル

WordPressの記事に著者情報カード（プロフィール写真・名前・肩書・経歴・SNSリンク）を自動挿入し、JSON-LD構造化データでE-E-A-Tシグナルを強化するSEOプラグインです。

Kashiwazaki SEO Author Schema Displayは、Googleが重視するE-E-A-T（経験・専門性・権威性・信頼性）の評価において重要な「誰がこのコンテンツを書いたのか」を、ユーザーと検索エンジンの両方に正確に伝えるために開発されました。記事に著者プロフィールカードを視覚的に表示するだけでなく、JSON-LD形式の構造化データとしてArticle / Person / Organization のスキーマをページのhead内に出力します。

著者カード表示

記事の指定位置に著者プロフィールカードを自動挿入。27種類の表示位置から選択可能

JSON-LDスキーマ

Article/Person/Organization の構造化データを自動出力。4つのスキーマモードに対応

E-E-A-T強化

Google検索でのE-E-A-Tシグナルを強化。著者の専門性・信頼性を検索エンジンに伝達

SEO / GEO における強み

著者情報の明示は、SEO・GEOの両面でコンテンツの信頼性評価に寄与します。本プラグインは視覚表示と構造化データの両方でこれを実現します。

JSON-LDによる構造化データ出力

本プラグインはページのhead内にJSON-LD形式の構造化データを出力します。Article / Person / Organization / Role の各スキーマタイプに対応しており、検索エンジンが著者情報を機械的に理解できます。これにより、Google検索結果でのリッチリザルト表示（著者名の表示等）の可能性が高まります。4つのスキーマモード（スキーマなし / シンプル著者 / 詳細著者 / 分離参照）から、サイトの構成に合った出力方式を選択できます。

E-E-A-Tの支援

Googleの品質評価ガイドラインにおいて、E-E-A-T（経験・専門性・権威性・信頼性）は重要な評価軸です。本プラグインは2つのレイヤーでE-E-A-Tを支援します。(1) 著者ボックスによる視覚的な著者情報の表示（ユーザー・品質評価者向け）と、(2) JSON-LDスキーマによる著者のRole・所属組織・プロフィールリンクの機械可読な出力（検索エンジン向け）です。特に「詳細著者」モードでは著者の役割（執筆者・監修者等）をスキーマに明示でき、YMYLジャンルでの信頼性評価に有効です。

GEO（生成エンジン最適化）への対応

AI検索（ChatGPT、Gemini、Perplexity等）がWebページを参照する際、構造化データに含まれる著者情報はコンテンツの信頼性判断の材料となります。JSON-LDで出力されたPerson / Organizationスキーマは、AIが「この記事は誰が書いたのか」を把握する際に参照できるデータです。また、記事内に視覚的に著者のプロフィール（肩書・経歴・SNSリンク）が明記されていることで、AIがそのコンテンツを専門家による情報として引用しやすくなる傾向があります。

軽量な実装

プラグインのフロントエンド出力はCSSのみで、外部JavaScriptの読み込みはありません。著者ボックスのHTMLとJSON-LDスクリプトタグをページに追加するだけのシンプルな実装のため、Core Web Vitals（LCP・CLS・INP）への悪影響がありません。
